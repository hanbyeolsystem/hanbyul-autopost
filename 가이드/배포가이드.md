# 한별 AI 백엔드 — 설치·연결 가이드

> 이 백엔드가 하는 일: 콘솔(브라우저)이 "사진+문구"를 보내면, 키를 안전하게 숨긴 채 실제 AI로 6채널 글·사진분석·그림·영상을 처리해 돌려줍니다.
> 핵심: **API 키는 이 서버에만 있고, 브라우저에는 절대 노출되지 않습니다.**

---

## 🔑 key.txt 설정 (한 줄에 키 하나씩, 필요한 것만)

`AI백엔드/key.txt` 파일에 아래처럼 넣으면 백엔드가 자동 인식합니다:

```
sk-ant-...        ← Claude 키 (글 생성 + 사진 분석). 필수
sk-...            ← OpenAI 키 (AI 그림 생성). 그림 쓸 때만
hf:힉스필드키      ← Higgsfield 키 (AI 영상 생성). 영상 쓸 때만, 앞에 hf: 붙이기
```

- Claude 키만 있으면: 글 생성 + 사진 분석까지 됨
- OpenAI 키 추가: 🎨 그림 생성 켜짐
- 힉스필드 키 추가(hf: 형식): 🎬 영상 생성 켜짐
- 키를 바꾸면 서버를 껐다 `실행.bat`으로 다시 켜야 적용됨

---

## 0. 폴더 구성

```
한별_AI백엔드/
├── server.js          ← 본체(의존성 없음, Node 18+면 그냥 실행됨)
├── Dockerfile         ← NAS 컨테이너용
├── docker-compose.yml ← 키 입력 + 실행 설정
└── 배포가이드.md       ← 지금 이 파일
```

---

## A. 가장 빠른 테스트 (내 PC에서, 5분)

NAS 가기 전에 PC에서 먼저 작동을 확인하는 길입니다.

1. Node.js 설치 (18 이상). 터미널에서 폴더로 이동.
2. 키를 환경변수로 넣고 실행:
   - Windows(PowerShell):
     ```
     $env:ANTHROPIC_API_KEY="sk-ant-..."; node server.js
     ```
   - Mac/Linux:
     ```
     ANTHROPIC_API_KEY="sk-ant-..." node server.js
     ```
3. 브라우저에서 `http://localhost:8787/health` 접속 → `{"ok":true,"ai":"anthropic",...}` 보이면 성공.
4. 콘솔(hanbyul-autopost-dashboard.html)의 **"AI 백엔드 주소"** 칸에 `http://localhost:8787` 입력 → 생성 버튼 → 진짜 AI 글이 나옵니다.

> 키가 없으면? `/health`에 `"ai":"none"`이 뜨고, 생성 시 콘솔이 자동으로 무료 공식 글로 폴백합니다(끊기지 않음).

## B. 시놀로지 NAS에 올리기 (실전 배포)

### B-1. 준비
- DSM에서 **Container Manager(구 Docker)** 패키지 설치.
- File Station으로 `한별_AI백엔드` 폴더를 NAS의 `/docker/hanbyul-ai`에 업로드.

### B-2. 키 입력
- `docker-compose.yml`을 열어 `ANTHROPIC_API_KEY` 값에 본인 Claude 키를 붙여넣기.
  (OpenAI를 쓰려면 그 줄을 주석 처리하고 `OPENAI_API_KEY` 줄을 살리세요.)

### B-3. 실행 (둘 중 하나)
- **방법 1 — Container Manager UI**: 프로젝트 생성 → 경로 `/docker/hanbyul-ai` 선택 → `docker-compose.yml` 인식 → 빌드/실행.
- **방법 2 — SSH**:
  ```
  cd /volume1/docker/hanbyul-ai
  sudo docker compose up -d --build
  ```

### B-4. 확인
- `http://NAS내부IP:8787/health` 접속 → `{"ok":true,...}` 확인.

### B-5. 콘솔 연결
- 콘솔의 "AI 백엔드 주소" 칸에 `http://NAS내부IP:8787` 입력.
- 외부(집/외근)에서도 쓰려면 DSM 역방향 프록시로 `https://post-ai.한별도메인.com` → `localhost:8787` 연결 권장(HTTPS).

## 🔒 보안 체크리스트 (가후의 당부)

- ✅ 키는 `docker-compose.yml`(서버)에만. 콘솔 HTML에는 절대 넣지 말 것.
- ✅ 운영 시 `ALLOW_ORIGIN`을 `*` 대신 콘솔 도메인으로 제한.
- ✅ 8787 포트를 인터넷에 직접 열지 말고, 역방향 프록시+HTTPS 뒤에 둘 것.
- ✅ `docker-compose.yml`(키 포함)은 외부 공유·깃 업로드 금지.
